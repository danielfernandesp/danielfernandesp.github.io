5 concepts and examples of DAX functions that are widely used in my daily

Part 2

Published by Daniel Fernandes on November 13th, 2023

Introduction

In this second post of the series "Concepts and examples of DAX functions widely used in daily life," I present 5 filter functions that I encounter daily, mainly in problems requiring combinations of different filters between fields in the same context. I have noticed that filter functions can be magical solutions, as laborious tasks like creating new relationships, fields, calculated fields in tables, and even new tables can be avoided. Below are the examples:

1. ALLEXCEPT:

Removes all filters from a table, except those specified in one or more columns.

Scenario: You want to calculate the sum of sales, removing all filters except the filter applied to the product column.

      
       TotalSalesExceptProduct =
       CALCULATE(
          SUM('Sales'[Value]),
          ALLEXCEPT('Sales', 'Products'[Name])
       )

In this example, ALLEXCEPT removes all filters from the 'Sales' table, except those applied to the 'Name' column of the 'Products' table.

2. ALLSELECTED:

Removes all filters from the tables in the context, except those applied explicitly.

Scenario: Calculate the average of sales, considering all selected products, regardless of other filters.

        
            AverageSelectedSales =
            CALCULATE(
               AVERAGE('Sales'[Value]),
               ALLSELECTED('Products')
            )

ALLSELECTED removes all filters from the tables, except the filters applied to the 'Products' table. It considers the selections made by the user in the other tables.

3. EARLIEST:

Returns the oldest value in a column, considering the applied filters.

Scenario: Get the date of the first sale for each product.

        
            FirstSaleDate =
            CALCULATE(
            EARLIEST('Sales'[Date]),
            ALLEXCEPT('Sales', 'Products'[Name])
            )

EARLIEST returns the oldest date within the provided context. Here, it is the date of the first sale for each product.

4. LOOKUPVALUE:

Searches for a value in a column and returns a corresponding value from another column in the same row.

Scenario: Get the price of the product from a price table.

        
         ProductPrice = LOOKUPVALUE('PriceTable'[Price], 'PriceTable'[ProductID], 'Products'[ID])

In this example, LOOKUPVALUE searches for a value in the specified column and returns a corresponding value from the same row in another column.

5. REMOVEFILTERS:

Removes all filters applied to the tables in the current context.

Scenario: Calculate the sum of sales, ignoring all applied filters.

        
         TotalSalesNoFilters =
         CALCULATE(
            SUM('Sales'[Value]),
            REMOVEFILTERS()
         )

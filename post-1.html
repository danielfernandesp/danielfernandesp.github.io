10 concepts and examples of DAX functions that are widely used in my daily

Introduction

Analysts in Business Intelligence (BI) who use tools like PowerBI, Excel Power Pivot, or Analysis Services Tabular, for example, need to have a command of the DAX language (Data Analysis Expressions). This language is used to create formulas and expressions that perform calculations on data. In everyday work, we encounter various coding scenarios where business rules tend to be complex, requiring a high level of reasoning from the developer analyst.

The only solution to such problems in these tools is often the development of complex, yet comprehensive, measures in DAX. Some functions may seem basic and routine, but when used correctly and nested, they are capable of solving anything from the simplest to the most complex business rule. Below, based on my own experience, I have listed concepts and examples of the 10 most useful functions in my DAX development routine.

1. SUMX: This function sums the values in a column after applying an expression. It is useful for creating summary measures.

Purpose: to sum the total value of the multiplication of Value and Quantity in each row of the table.

                    
                        SUMX(Table, Table[Value] * Table[Quantity])

2. AVERAGE: Calculates the average of values in a column.

Purpose: calculates the average of grades in a table.

                    
                        AVERAGE(Table[Grades])

3. COUNT: Counts the number of rows in a table or column that contain data.

Purpose: counts the number of customers in a table.

                    
                        COUNT(Table[Customers])

4. MAX and MIN: Finds the maximum and minimum value in a column, respectively.

Purpose: finds the maximum value of sales in a table.

                    
                        MAX(Table[Sales])

5. FILTER: Filters the data of a table based on a specified condition.

Purpose: filters the table to include only rows where sales are greater than 1000.

                    
                        FILTER(Table, Table[Sales] > 1000)

6. CALCULATE: Modifies the evaluation context for an expression, allowing you to apply filters or conditions.

Purpose: calculates the sum of sales only for the year 2023.

                    
                        CALCULATE(SUM(Table[Sales]), Table[Year] = 2023)

7. RANKX: Calculates the rank of a value relative to other values in a column.

Purpose: ranks sales in descending order.

                    
                        RANKX(Table, Table[Sales], , DESC, Dense)

8. RELATED: Returns a related value from a related table.

Purpose: retrieves related information from another table based on the relationship.

                    
                        RELATED(RelatedTable[Information])

9. EARLIER: Returns a value from the previous row during a calculation iteration.

Purpose: retrieves the value from the previous row during a calculation.

                    
                        EARLIER(Table[Value])

10. SWITCH: Acts as a switch/case statement in DAX, allowing for different actions based on conditions.

Purpose: assigns labels based on sales.

                    
                        SWITCH(TRUE(), Table[Sales] > 1000, "High", Table[Sales] > 500, "Medium", "Low")
